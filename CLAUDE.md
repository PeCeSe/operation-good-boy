## What This Is

A co-op browser-based deck-builder for 2–4 players. Players are cats working together to defeat Good Boy (the final enemy) before the neighborhood is overrun with cucumbers. Built with React + Vite (frontend) and Node.js + Socket.IO (backend).

---

## Tech Stack

- **Frontend**: React, Vite, Tailwind CSS, Socket.IO client
- **Backend**: Node.js, Express, Socket.IO
- **Hosting**: Frontend on Vercel (auto-deploys from `main`), backend on Render (manual redeploy required)
- **Repo structure**: Monorepo — `frontend/` and `backend/` as separate apps

## Core Mechanics (Agreed)

### Turn Flow
Each player's turn starts with a **phase overlay** sequence:
1. **Turn start card** — announces whose turn it is, auto-revealed
2. **Event cards** — drawn from the event deck, flipped face-down, player clicks to reveal, then applies effect
3. **Enemy ability cards** — enemies with `trigger: "start_of_round"` show their ability, player applies it

Events and enemy abilities are drawn **per turn** (not per round) — every player gets their own events at the start of their turn. This was an intentional decision so all players experience the reveal flow, not just the first player of each round.

### Discard Mechanic
When an event or enemy ability requires discarding cards, players **choose** which cards to discard via a dedicated overlay. Cards can be dragged to a discard zone or clicked. The phase is paused until all affected players have discarded. The phase then auto-resumes.

### Locations & Cucumbers
- 3 locations, played sequentially
- Each location has a `maxCucumberTokens` limit
- Reaching max triggers **immediate location loss** (not at end of round) — the next location activates, or defeat if none remain
- Defeat condition: all locations lost

### Victory Condition
Defeat all enemies including **Good Boy** (always at the bottom of the enemy deck). When the enemy deck and board are empty, players win.

### Stun
When a player reaches 0 lives:
- They are stunned (skip their next turn recovering)
- Discard half their hand
- Add 1 cucumber to current location
- On their stunned turn: recover to full lives, unstunned

### Attack Types
`scratch`, `bite`, `ignore`, `charm` — enemies have weaknesses and resistances. Weak = 2× damage, resistant = ½ damage.

### Shop
6 cards visible at a time, refilled from shop deck at end of each turn. Cards have a pawcoin cost. Shop can be blocked by events.

### Pawcoins
Generated by playing cards. Reset to 0 at end of turn. Can be reduced by event `pawcoinPenalty`.

## State Architecture

### Backend (`backend/src/game/`)
- `gameState.js` — initializes game state
- `turnLogic.js` — all game logic (startRound, revealPhase, advancePhase, endTurn, discardCards, etc.)
- `roomManager.js` — socket-to-room mapping, rejoin logic
- `data/` — cards, characters, enemies, events, locations

### Key State Fields
```
pendingPhase: { items, resolvedIndex, currentRevealed } | null
pendingDiscard: [{ playerId, count }] | null
```

- `pendingPhase` controls the card-reveal overlay between turns
- `pendingDiscard` pauses the phase until all players choose cards to discard
- All game actions (play card, attack, buy, end turn) are blocked while either is set

### Frontend (`frontend/src/`)
- `pages/Game.jsx` — main game screen, owns `effectResult` state (computed from gameState diffs)
- `pages/Lobby.jsx` — character selection and ready-up
- `pages/Home.jsx` — create/join room
- `components/PhaseOverlay.jsx` — card-reveal overlay
- `components/DiscardOverlay.jsx` — discard selection overlay
- `components/PlayerBoard.jsx` — the current player's full board (collapsible character info, hand, attacks, shop)
- `data/characters.js` — shared character data (names, images, colors) used by both Lobby and Game

## What Is Temporary / Not Finished

- **Pacing and flow**: Still being refined. The per-turn event reveal flow is new and hasn't been extensively playtested.
- **Card content**: All card names, flavor text, and effects are placeholder. Final card design pass not done.
- **Event content**: Events are mostly placeholder — names and effects need a full content pass.
- **Enemy content**: Enemy stats, abilities, and rewards are placeholder.
- **Design and colors**: Current UI is functional but unstyled in many places. A full design pass (colors, typography, spacing, visual polish) is planned for later.
- **Sound and animation**: Minimal animations exist. No sound effects yet.
- **Card artwork**: Cards show type emoji placeholders. Real card art is not designed yet.
- **4th character**: Slot exists in the game but no character is designed for it yet.
- **Win/defeat screens**: Exist but are basic. Need visual polish.
- **Mobile layout**: Works but not optimized for small screens.

## Important Notes for Claude

- **Frontend deploys automatically** when pushing to `main` via Vercel.
- **Backend must be manually redeployed** on Render after backend changes.
- The `CHARACTERS` array in `frontend/src/data/characters.js` is the single source of truth for character display data (names, images, colors). Backend character data lives in `backend/src/data/characters.js`.
- `endRound` always resets `currentPlayerId` to `players[0]` (the first player who joined, usually the host) at the start of each new round.
- Do not auto-discard cards — always use `pendingDiscard` so players choose.
- Check location loss immediately after any cucumber addition (`checkLocationLoss`), not only at end of round.
